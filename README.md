## 게임 소개

**픽셀 물풍선 대전**은 2명의 플레이어가 같은 맵에서 물풍선을 설치해 상대를 제거하는 대전 게임입니다.  
물풍선은 일정 시간 후 터지며, 터진 물줄기는 십자 방향으로 퍼져나갑니다.  
물줄기에 갇히면 즉시 사망하지 않고 일정 시간 안에 바늘 아이템으로 탈출할 수 있습니다.

## 실행 방법

1. `ConsoleBnB.exe` 파일을 실행합니다.
2. 콘솔 창과 함께 **게임 그래픽 창**이 자동으로 열립니다.
3. 콘솔 창은 로그용이며, 실제 게임은 **그래픽 창**에서 진행됩니다.
4. 그래픽 창이 활성화된 상태에서 키보드로 조작합니다.

> ⚠️ `font_korean.png` 파일이 `.exe`와 같은 폴더에 있어야 한글이 정상 출력됩니다.

## 게임 화면 구성

### 메인 메뉴 화면

```
┌─────────────────────────────────┐
│         픽셀 물풍선 대전          │
│      [ 시작 ]                    │
│      [ 도움말 ]                  │
│      [ 나가기 ]                  │
│  클릭 또는 방향키와 엔터          │
└─────────────────────────────────┘
```

- **시작**: 게임을 시작합니다.
- **도움말**: 게임 방법 화면을 표시합니다.
- **나가기**: 게임을 종료합니다.

### 게임 플레이 화면

```
┌──────────────────────────────────────┐
│ [1P] 폭탄 3  사거리 2  바늘 0        │
│              대전             [2P]    │
├──────────────────────────────────────┤
│                                      │
│         [ 17 × 13 게임 맵 ]          │
│                                      │
├──────────────────────────────────────┤
│ 1P 생존  WASD / Shift   방향키 / Space  2P 생존 │
└──────────────────────────────────────┘
```

- **상단**: 각 플레이어의 폭탄 개수, 사거리, 바늘 보유 수 표시
- **중앙**: 17×13 크기의 게임 맵
- **하단**: 플레이어 상태(생존/갇힘/탈락) 및 조작키 안내

### 결과 화면

게임 종료 시 다음 중 하나가 표시됩니다:

- `1P 승리` — 2P가 탈락했을 때
- `2P 승리` — 1P가 탈락했을 때
- `무승부` — 두 플레이어가 동시에 탈락했을 때
- `게임 종료` — ESC로 강제 종료했을 때

## 게임 규칙

### 기본 규칙

1. 1P는 맵 좌상단 `(1, 1)`, 2P는 맵 우하단 `(15, 11)`에서 시작합니다.
2. 물풍선은 설치 후 **60프레임(약 3초)** 뒤 자동으로 폭발합니다.
3. 폭발한 물풍선은 **십자 방향으로 물줄기**를 사거리만큼 퍼뜨립니다.
4. 물줄기는 **8프레임** 동안 유지된 후 사라집니다.

### 이동 규칙

- 플레이어는 상하좌우 한 칸씩 이동합니다.
- 이동 속도는 기본 **7프레임당 1칸**입니다.
- 속도 아이템 획득 시 **4프레임당 1칸**으로 빨라집니다.
- 상대 플레이어가 있는 칸으로 이동할 수 없습니다.
  - 단, 상대가 `갇힘` 상태일 경우 해당 칸으로 이동하면 상대가 즉시 사망합니다.
- 물풍선이 설치된 칸으로 이동할 수 없습니다 (물풍선이 벽 역할).

### 물풍선 규칙

- 플레이어가 서 있는 칸에 물풍선이 설치됩니다.
- 설치 후 플레이어는 해당 칸을 벗어날 수 있습니다.
- 동시에 설치 가능한 물풍선 수는 `maxBombs`에 따라 결정됩니다 (기본 3개).
- 물줄기가 다른 물풍선에 닿으면 **연쇄 폭발**이 발생합니다.
- 물줄기는 **벽(Wall)** 을 통과하지 못합니다.
- 물줄기가 **상자(Block)** 에 닿으면 상자가 파괴되고, 아이템이 등장할 수 있습니다.

### 갇힘 & 탈출 규칙

- 물줄기에 닿은 플레이어는 즉시 사망하지 않고 **갇힘(trapped)** 상태가 됩니다.
- 갇힘 상태에서 **100프레임(약 5초)** 이 지나면 사망합니다.
- 갇힘 상태에서 **바늘(needle)** 을 보유한 경우 `N` 키를 눌러 즉시 탈출할 수 있습니다.
- 탈출 시 바늘 보유 수가 1 감소합니다.
- 갇힘 상태에서는 이동 및 물풍선 설치가 불가합니다.

### 승패 판정

| 상황 | 결과 |
|------|------|
| 1P만 생존 | 1P 승리 |
| 2P만 생존 | 2P 승리 |
| 둘 다 사망 | 무승부 |

## 아이템 설명

상자가 파괴되면 아이템이 등장할 수 있습니다.  
플레이어가 아이템 위를 지나가면 자동으로 획득됩니다.

### 아이템 드롭 확률 시스템

상자가 부서질 때 **2단계 주사위** 방식으로 아이템을 결정합니다.

**1단계 — 드롭 여부 결정 (30% 확률)**
```
1~100 사이 난수 생성 → 30 이하면 아이템 드롭 당첨
                      → 31 이상이면 아이템 없이 빈 칸만 남음
```

**2단계 — 아이템 종류 결정 (각 25% 균등 확률)**
```
0~3 사이 난수 생성
  0 → 초록 번개 (Speed)
  1 → 분홍 플러스 (Range)
  2 → 파란별 (BombCount)
  3 → 노란 바늘 (Needle)
```

> 난수 생성기: 메르센 트위스터(`std::mt19937`) 사용 — 게임 시작 시 `std::random_device`로 시드 초기화

### 아이템 효과

| 아이템 | 색상 | 효과 | 제한 |
|--------|------|------|------|
| 파란별 (BombCount) | 파란색 | 최대 물풍선 설치 수 +1 | 최대 8개 |
| 분홍 플러스 (Range) | 분홍색 | 물줄기 사거리 +1 | 최대 8칸 |
| 초록 번개 (Speed) | 초록색 | 이동 속도 증가 (4프레임/칸) | 100프레임 지속 |
| 노란 바늘 (Needle) | 노란색 | 바늘 보유 수 +1 | 최대 3개 |

## 한글 폰트 UV 매핑 방식

게임 내 한글 출력은 **비트맵 폰트 아틀라스(font_korean.png)** 방식을 사용합니다.  
TTF 폰트 대신 PNG 이미지에 모든 글자를 격자로 배열해두고, 필요한 글자만 잘라서 화면에 그립니다.

### font_korean.png 구조

```
┌────┬────┬────┬────┬ ··· ┬────┐  ← 1행 (16칸)
│ !  │ "  │ #  │ $  │     │ /  │  ASCII 특수문자 (index 0~)
├────┼────┼────┼────┼ ··· ┼────┤
│ 0  │ 1  │ 2  │ 3  │     │ ?  │  숫자 0~9, 기호
├────┼────┼────┼────┼ ··· ┼────┤
│ @  │ A  │ B  │ C  │     │ O  │  대문자 A~Z
├────┼────┼────┼────┼ ··· ┼────┤
│ P  │ Q  │ R  │ S  │     │ _  │
├────┼────┼────┼────┼ ··· ┼────┤
│ `  │ a  │ b  │ c  │     │ o  │  소문자 a~z
├────┼────┼────┼────┼ ··· ┼────┤
│ p  │ q  │ r  │ s  │     │ 가 │  한글 시작
├────┼────┼────┼────┼ ··· ┼────┤
│ 각 │ 갇 │ 개 │ 거 │     │ 기 │  한글 계속...
│                  ···          │
│ 항 │ 홍 │    │    │     │    │  한글 끝 + 자모
└────┴────┴────┴────┴ ··· ┴────┘
```

### 매핑 규칙

| 항목 | 값 |
|------|-----|
| PNG 크기 | 576 × 756 px |
| 셀(글자 1칸) 크기 | 36 × 36 px |
| 열 수 (COLS) | 16 |
| 문자 순서 | ASCII(32~126) → 한글 → 자모 |

### 글자 → 좌표 변환 공식

```cpp
// 1. 글자의 인덱스 계산
if (ASCII 문자)       index = (문자코드) - 32          // 32(' ') ~ 126('~')
if (한글)             index = 95 + koreanGlyphs_.find(ch)
if (자모)             index = 95 + 한글수 + jamoGlyphs_.find(ch)

// 2. PNG에서 잘라낼 좌표 계산 (UV 매핑)
sourceX = (index % 16) * 36   // 열 위치
sourceY = (index / 16) * 36   // 행 위치

// 3. GDI+로 해당 영역(36×36)만 잘라서 화면에 그리기
graphics.DrawImage(atlas, destRect, sourceX, sourceY, 36, 36, UnitPixel)
```

### 색상 변경 방식

PNG는 흰색 글자로 저장되어 있으며, GDI+의 **ColorMatrix(색상 행렬)** 를 이용해  
렌더링 시점에 원하는 색상으로 실시간 변환합니다.

```cpp
// 예: 빨간색(RGB 239,111,108)으로 글자 출력 시
ColorMatrix tint = {
    239/255.0f, 0, 0, 0, 0,   // R 채널
    0, 111/255.0f, 0, 0, 0,   // G 채널
    0, 0, 108/255.0f, 0, 0,   // B 채널
    0, 0, 0,          1, 0,   // Alpha 유지
    0, 0, 0,          0, 1
};
```
